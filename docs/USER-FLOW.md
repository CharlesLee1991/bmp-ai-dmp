## 0. 한눈에 — 두 가지 사용자 흐름

DMP에서 오디언스를 만들어 런컴으로 송출하는 경로는 **2가지**다.

```
[흐름 1] 정의형 — 페르소나·필터로 골라 담아 확정
  페르소나(정의) ─┐
  카드/지하철/버스/ │  🛒 담기   ┌──────────┐  타겟으로   ┌────────────┐  송출  ┌────────┐
  멤버십 탐색      ─┼─────────▶ │ 오디언스   │  저장      │ 생성된      │──────▶│ 전송   │
  (필터 조건)      │           │ 카트(드로어)│──────────▶ │ 오디언스(허브)│       │ 이력   │
                  │           └──────────┘  (라벨·태그  └────────────┘       └────────┘
                  │                          ·메모)
[흐름 2] 생성형 — 자연어 한 번으로 AI가 생성
  퀵 AI 오디언스 ──────── 승인 → BQ 오디언스 테이블 생성 ──────────────────┘
  (NL→SQL→승인)          "생성된 오디언스로 이동 →" CTA
```

- **흐름1(정의형)**: 사용자가 페르소나/필터로 조건을 **골라 카트에 담고**, 이름·라벨·태그·메모를 붙여 **타겟으로 확정**한 뒤 **생성된 오디언스** 허브에서 정리·송출.
- **흐름2(생성형)**: **퀵 AI 오디언스**에서 문장 한 번으로 정의~생성까지 끝내고, 생성물이 **생성된 오디언스** 허브에 자동 합류.
- 두 흐름은 **생성된 오디언스 허브**에서 합류하며, 송출·전송이력으로 이어진다.

## 1. 메뉴 구조 (사이드바)

업무 흐름 순서대로 그룹화(`components/DmpSidebar.tsx`).

| 그룹 | 탭 | 역할 |
|---|---|---|
| **개요** | 대시보드(홈) | 운영 현황 오버뷰 + 사용자 흐름 가이드 + **시작하기**(페르소나로 시작 / 퀵 AI 오디언스로 시작) 2버튼. 기본 진입 화면 |
| **1 · 타겟 정의** | 페르소나 | 페르소나(별명+필터세트+AI 라이프스타일) 정의·저장 |
| **2 · 오디언스 탐색** | 카드 · 지하철 · 버스 · 멤버십 | 데이터소스별 브라우징·필터 정제 |
| **3 · 추출 · 전송** | 퀵 AI 오디언스 · 생성된 오디언스 · 전송 이력 | AI 생성 · 타겟 관리/송출 · 송출 기록 |
| **4 · 성과 · 인사이트** | 매체 성과 · 소비 트렌드 · 카드사 비교 · 쇼핑상품 | 폐루프 성과 + 시장 인사이트 |
| **시스템 관리** | 분류 맵핑 관리 | 코드→라벨 매핑 등 운영(admin) |

- 헤더는 전 화면 고정: 브레드크럼(`DMP Explorer › RUNCOMM › {그룹} › {메뉴}`) + LIVE + **🛒 카트 버튼(수량 배지)** + 테마.
- 상단 '타겟 정의' 도킹바(1차 모수 + 페르소나 칩 + 공통 데모그래픽)는 **탐색(explore) 탭에서만** 노출.

## 2. 단계별 상세

### 2-1. 타겟 정의 — 페르소나 (`PersonaStudio.tsx`)
- 좌: 별명·자연어(AI 3안 라벨칩)·수동 필터칩 / 우: 라이브 프리뷰(모수·연령성별·지역/업종 TOP·AI 라이프스타일).
- **AI 라벨칩 클릭** → 필터 적용 + **페르소나 별명 자동 반영**(모수 값 제외).
- **핸드오프**:
  - "저장 후 **탐색 화면에 적용**" → 필터를 화면에 주입하고 **카드 탭으로 자동 이동**.
  - 목록 행의 **🛒 담기** → 페르소나를 바로 카트에 담기.

### 2-2. 오디언스 탐색 (카드 / 지하철·버스 / 멤버십)
- 각 화면 상단 액션: **AI 타겟 도우미 · 오디언스 담기 · 런컴 타겟 전송**.
- **담기 진입점**: explore 헤더 "오디언스 담기" → 이름 미니폼 → 카트.
- 페르소나 칩 🛒로도 담기.
- ⚠️ **EF 추출 한계(중요)**: 담기·송출은 아래 축만 실제 추출된다.
  `sex · age_group · region · (city·district) · major/middle/subcategory · shop_category · upload_session · cat1 · apprl`
  → **지하철 역·노선, 멤버십 가맹점·적립앱·요일·금액구간은 추출 불가(탐색 전용)**. 담기 시 "탐색 전용, 데모그래픽만 반영"으로 정직 표기.

### 2-3. 오디언스 카트 (`CartDrawer.tsx`)
- 담긴 조각(필터/페르소나/AI 테이블) 확인·제거·합계.
- **메인 액션 = "타겟으로 저장"**(생성된 오디언스에 보관), **서브 = "바로 송출"**(저장 겸용). 두 버튼에 카드형 Radix 툴팁.
- 저장/송출 후 **"생성된 오디언스로 이동"** 버튼 제공.
- 조각별 라벨·태그·메모 지정(접이식).

### 2-4. 퀵 AI 오디언스 (`AiExploreTab.tsx`)
- 자연어 → AI가 SQL 생성 → 인원 조회(선택) → **승인 시에만** BQ 오디언스 테이블 생성(30일 보관). 페르소나 불필요.
- **핸드오프**: 생성 완료 + 이력 헤더에 **"생성된 오디언스에서 보기 →"** CTA(탭 이동).

### 2-5. 생성된 오디언스 (통합 허브) (`TargetAudienceTab.tsx`)
- **2소스 통합**: ① 카트 확정본(필터/페르소나) ② AI 생성본(퀵 AI BQ 테이블).
- **리스팅**: 타겟명·분류 라벨·태그·메모·**생성자**·상태·예상 모수.
- **필터**: 소스 / **라벨 드롭다운(내부 검색·건수, 캠페인·고객명 용도)** / 태그 다중 / 상태 / 생성자 / 검색 / **페이징**.
- **관리**: 송출(dev·prod 모달) · 복제 · 병합(합집합·중복제거) · 편집(라벨·태그·메모) · 삭제 · 카트로 불러오기 · (AI 생성본) 카트에 담기.
- 'saved' 송출 성공 시 **`send_count`·`last_sent_at` 기록**(상태는 재사용 위해 유지) → 행에 "송출 N회·최근 송출일" 배지.
- 권한: admin=전체 저장/송출본, advertiser=본인. 편집/삭제는 소유자·admin.

### 2-6. 전송 이력 (`ExportHistoryTab.tsx`)
- `/api/exports`(`de_dmp_export_history`) 조회. 그룹명·필터요약·오디언스수·환경·전송자·일시·런컴ID.
- 카트 묶음 송출은 `memo="카트 묶음: {이름} ({조각})"`로 표시.

## 3. 데이터·연동 (요약)

| 대상 | 저장/연동 |
|---|---|
| 페르소나 | `de_dmp_personas` (`/api/personas`) |
| 카트·타겟 묶음 | `de_dmp_audience_carts` (`/api/carts`) — status: cart/saved/submitted, label·tags·memo·user_name·send_count·last_sent_at |
| AI 생성 오디언스 | data-worker `/dmp/segment/explore` (`/api/ai-explore`) → BQ 테이블 |
| 라벨 오버라이드 | `de_dmp_label_overrides` (`/api/label-overrides`) |
| 송출 | Edge Function `dmp-target-export` (필터 모드 / `bq_audience_table` 모드) → 런컴 |
| 전송 이력 | `de_dmp_export_history` (`/api/exports`) |

## 5. ⚠️ 미검증/후속 (개발 주의)

- **실제 송출(런컴 전송) 기능은 별도 검증·구현 대상.** 현재 프론트는 EF `dmp-target-export` 호출 및 이력 기록까지 배선돼 있으나, **엔드투엔드 실송출(ADID 추출 → S3 → 런컴 API 수신 → 결과 회수)의 실검증은 별도 진행 필요**. dev/prod 게이트·상용 승인 절차 포함.
- 카트 '바로 송출'은 조각별 개별 타겟(경로 A). **ID 레벨 중복제거(1묶음=1타겟) 통합 송출은 EF v21(후속)**.
- 인라인 `AiExplore.tsx`(카드탭 패널) vs 메뉴 `AiExploreTab.tsx` 병존 — 정리 후보.
