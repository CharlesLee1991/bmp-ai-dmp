## 1. 시작하기

### 접속과 로그인
1. 브라우저에서 **https://dmp.bmp.ai** 접속
2. 발급받은 **아이디 / 비밀번호**로 로그인

> 계정 발급·비밀번호 재설정은 BizSpring 담당자에게 문의해 주세요.

### 계정 권한
| 권한 | 볼 수 있는 범위 |
|---|---|
| **관리자(admin)** | 전 메뉴 + 성과·인사이트 + 시스템 관리. 모든 사용자의 타겟 열람 |
| **광고주(advertiser)** | 타겟 정의 · 오디언스 탐색 · 추출/전송. **본인이 만든 타겟만** 열람 |

> 권한에 따라 좌측 메뉴 구성이 달라집니다. 화면에 보이지 않는 메뉴가 있다면 담당자에게 권한을 문의해 주세요.

### 권장 환경
- Chrome / Edge / Safari 최신 버전
- 화면 해상도 **1440px 이상** 권장 (일부 화면은 좌우 2단 구성)
- 라이트 / 다크 테마는 우측 상단에서 전환할 수 있습니다.

## 2. 화면 구성 — 사이드바 5그룹

메뉴는 **업무 흐름 순서**대로 좌측 사이드바에 배치되어 있습니다.

```
┌─────────────┬────────────────────────────────────────────────┐
│ DMP Explorer│ DMP Explorer › RUNCOMM › 오디언스 탐색 › 카드   │ ← 브레드크럼
│             │            ● LIVE    🛒 카트(수량)    테마      │ ← 헤더(고정)
│ 1·타겟 정의 ├────────────────────────────────────────────────┤
│   페르소나  │ 1차 타겟 모수 · 페르소나 칩 · 공통 필터(성별/연령/지역) │ ← 타겟 정의 도킹
│             ├────────────────────────────────────────────────┤
│ 2·오디언스  │ [AI 타겟 도우미]   [🛒 오디언스 담기]   [🚀 전송] │ ← 액션 3존
│    탐색     ├────────────────────────────────────────────────┤
│   카드      │ 2차 필터 — 업종·쇼핑·매체·금액·통신·소비·결제…   │
│   지하철    ├────────────────────────────────────────────────┤
│   버스      │  이용자 수  │  남녀 비율  │  주력 연령대        │ ← 요약 카드
│   멤버십    ├────────────────────────────────────────────────┤
│             │  업종 TOP12 │ 연령×성별 분포 │ 지역별 이용자    │ ← 분석 영역
│ 3·추출·전송 │                                                │
│   퀵 AI     │                                                │
│   생성된 O. │                                                │
│   전송 이력 │                                                │
│             │                                                │
│ 4·성과·인사이트                                              │
│   매체 성과 │                                                │
│   소비 트렌드                                                │
│   카드사 비교                                                │
│   쇼핑상품  │                                                │
│             │                                                │
│ 시스템 관리 │                                                │
│   분류 맵핑 │                                                │
└─────────────┴────────────────────────────────────────────────┘
```

| 그룹 | 메뉴 | 하는 일 |
|---|---|---|
| **1 · 타겟 정의** | 페르소나 | 별명 + 필터 세트를 하나의 페르소나로 정의·저장 |
| **2 · 오디언스 탐색** | 카드 / 지하철 / 버스 / 멤버십 | 데이터소스별로 살펴보며 조건 좁히기 |
| **3 · 추출 · 전송** | 퀵 AI 오디언스 / 생성된 오디언스 / 전송 이력 | AI 생성 · 타겟 관리와 송출 · 송출 기록 |
| **4 · 성과 · 인사이트** | 매체 성과 / 소비 트렌드 / 카드사 비교 / 쇼핑상품 | 집행 성과와 시장 인사이트 *(관리자 전용)* |
| **시스템 관리** | 분류 맵핑 관리 | 업종 코드 → 표시 라벨 지정 *(관리자 전용)* |

**필터는 3계층입니다.**
1. **타겟 정의 도킹**(상단 고정) — 1차 타겟 모수, 페르소나 칩, 공통 데모그래픽(성별·연령·지역). 탐색 화면에 공통 적용됩니다.
2. **2차 필터** — 화면별 전용 조건 (카드 = 업종·금액·매체 등)
3. **화면 내 세부 옵션** — 기간, 정렬 등

## 3. 두 가지 사용 흐름

오디언스를 만들어 런컴으로 보내는 경로는 **2가지**입니다.

```
[흐름 1] 정의형 — 골라서 담아 확정
  페르소나 ─┐
            ├─ 🛒 담기 ─→ 오디언스 카트 ─ 타겟으로 저장 ─→ 생성된 오디언스 ─ 송출 ─→ 전송 이력
  탐색 필터 ─┘

[흐름 2] 생성형 — 문장 한 줄로 AI가 생성
  퀵 AI 오디언스 ── 승인 ──→ 오디언스 자동 생성 ───────────→ 생성된 오디언스 ─ 송출 ─→ 전송 이력
```

- **흐름 1**은 조건을 직접 고르는 방식, **흐름 2**는 AI에게 맡기는 방식입니다.
- 두 흐름 모두 **생성된 오디언스** 허브에서 만나며, 여기서 정리하고 송출합니다.

## 4. 타겟 정의 — 페르소나

**페르소나 = 별명 + 필터 세트.** 자주 쓰는 타겟 조건을 저장해 두고 재사용합니다.

### 만드는 법
1. **1 · 타겟 정의 › 페르소나** 진입
2. **별명** 입력 (예: `강남 뷰티 2030`)
3. 조건 지정 — 둘 중 편한 방식으로:
   - **자연어 입력** → AI가 라벨 3안 제시 → **라벨칩 클릭 시 필터 자동 적용 + 별명 자동 반영**
   - **수동 필터칩** 직접 선택
4. 우측 **라이브 프리뷰**로 즉시 확인 — 모수 / 연령·성별 / 지역·업종 TOP / AI 라이프스타일 요약

### 저장 후 이어가기
| 버튼 | 동작 |
|---|---|
| **저장 후 탐색 화면에 적용** | 필터를 적용한 채 **카드 화면으로 자동 이동** |
| 목록의 **🛒 담기** | 페르소나를 바로 카트에 담기 |

> 페르소나는 서버에 저장되어 다음 접속 때도 그대로 남아 있습니다.

## 5. 오디언스 탐색 — 카드·지하철·버스·멤버십

각 화면 상단에 **액션 3존**이 있습니다.

| 버튼 | 하는 일 |
|---|---|
| **AI 타겟 도우미** | 목적을 말하면 조건을 추천 → `이 조건으로 탐색` 클릭 시 필터 자동 적용 |
| **🛒 오디언스 담기** | 현재 조건에 이름을 붙여 카트에 담기 |
| **🚀 런컴 타겟 전송** | 현재 조건으로 바로 전송 |

### 화면별 데이터
| 화면 | 내용 |
|---|---|
| **카드** | 카드 소비 — 업종·지역·금액·매체(카드사)·통신·단말·결제 조건 (기본 화면) |
| **지하철** | 지하철 이용 패턴 — 역·승하차·시간대 |
| **버스** | 버스 이용 패턴 |
| **멤버십** | 멤버십 제휴사·적립앱·시간대·요일·금액대 |

### 주요 필터 (카드 화면 기준)
| 필터 | 설명 |
|---|---|
| 성별 / 연령 / 지역 | 상단 **타겟 정의 도킹**에서 공통 적용 (지역: 시도 → 시군구 → 읍면동) |
| 업종 | 대분류(13) → 중분류 → 소분류 |
| 쇼핑 | 쇼핑 카테고리(14) |
| 금액 | 금액 구간 또는 상위/하위 10·20·30% |
| 매체 | 매체·카드사(10) |
| 통신 / 단말 | 통신사, 단말 종류 |
| 소비 | 소비 대카테고리(13) |
| 결제 | 승인횟수 / 승인금액 + 기간(최근 90일 등) + 조건(이상·이하) |
| 기간 | 1개월 / 3개월 / 6개월 / 1년 / 직접 선택 |
| **ADID** | 보유 광고식별자 파일 업로드 → 매칭 |

**ADID 업로드**: `+ ADID 파일 업로드` → 파일 선택 → 매칭 결과 표시 (`12,345건 매칭 / 20,000건 업로드 · 61.7%`). 업로드 목록과 DMP 데이터가 겹치는 대상만 매칭됩니다.

### ⚠️ 담기·전송에 반영되는 조건 (중요)
탐색 화면에서는 모든 조건을 볼 수 있지만, **실제 대상자 추출에 반영되는 축은 아래로 한정**됩니다.

> 성별 · 연령 · 지역(시도·시군구·읍면동) · 업종(대/중/소) · 쇼핑 카테고리 · ADID 업로드분 · 관심 카테고리 · 결제 조건

**지하철의 역·노선, 멤버십의 가맹점·적립앱·요일·금액구간은 탐색 전용**이며 추출에는 반영되지 않습니다(성별·연령·지역만 반영). 해당 화면에서 담기를 하면 안내가 표시됩니다.

## 6. 오디언스 카트 — 담고, 묶고, 이름 붙이기

장바구니처럼 **여러 조건을 담아 하나의 타겟으로 묶는** 기능입니다.

### 사용법
1. 탐색 화면에서 **🛒 오디언스 담기** → 이름 입력 → 카트에 추가
   (페르소나 목록의 🛒, 퀵 AI 생성물도 담을 수 있습니다)
2. 헤더의 **🛒 카트 버튼**(수량 배지) 클릭 → 카트 열기
3. 담긴 조각 확인 — 개별 제거 / 합계 확인
4. 조각별 **라벨 · 태그 · 메모** 지정 (접이식)
5. 마무리:

| 버튼 | 동작 | 권장 |
|---|---|---|
| **타겟으로 저장** *(메인)* | 생성된 오디언스에 보관 — 나중에 정리·재사용·송출 | ⭐ |
| 바로 송출 *(서브)* | 저장과 동시에 즉시 전송 | 급할 때만 |

6. 완료 후 **`생성된 오디언스로 이동 →`** 버튼으로 이어집니다.

> **라벨**은 캠페인명·고객명 등 내부 분류용으로 쓰면 편리합니다. 허브에서 라벨로 골라볼 수 있습니다.

## 7. 퀵 AI 오디언스 — 문장 한 줄로 생성

페르소나나 필터 없이, **문장 하나로 정의부터 생성까지** 끝냅니다.

```
자연어 입력 → AI가 조건 생성 → 인원 미리보기(선택) → 승인 → 오디언스 생성
```

1. **3 · 추출·전송 › 퀵 AI 오디언스** 진입
2. 찾고자 하는 대상을 문장으로 입력
3. AI가 조건을 만들고, **인원 사전 조회**로 규모를 먼저 확인할 수 있습니다
4. **승인해야만** 실제 오디언스가 생성됩니다 (승인 전에는 아무것도 만들어지지 않습니다)
5. 생성된 오디언스는 **30일간 보관**되며, `생성된 오디언스에서 보기 →` 로 이동합니다

## 8. 생성된 오디언스 — 통합 허브

**두 흐름의 결과물이 모두 모이는 곳**입니다.

- ① 카트에서 확정한 타겟 (필터·페르소나)
- ② 퀵 AI로 생성한 오디언스

### 리스팅 항목
타겟명 · 분류 라벨 · 태그 · 메모 · **생성자** · 상태 · 예상 모수 · 송출 배지

### 찾기
소스 / **라벨 드롭다운**(건수 표시) / 태그 다중 / 상태 / 생성자 / 검색어 / 페이징

### 관리 기능
| 기능 | 설명 |
|---|---|
| **송출** | 런컴으로 전송 (테스트·운영 선택 모달) |
| **복제** | 기존 타겟을 복사해 변형 |
| **병합** | 여러 타겟을 합집합으로 합치기 (중복 제거) |
| **편집** | 라벨·태그·메모 수정 |
| **카트로 불러오기** | 다시 카트에 담아 재구성 |
| **삭제** | 제거 (소유자 또는 관리자만) |

> 송출에 성공하면 **송출 N회 · 최근 송출일** 배지가 표시되고, 타겟은 재사용을 위해 그대로 유지됩니다.
> 광고주 권한은 본인 타겟만, 관리자 권한은 전체를 볼 수 있습니다.

## 9. 타겟 전송 · 전송 이력

### 전송 절차
1. 조건 설정 또는 오디언스 선택
2. **🚀 런컴 타겟 전송** (또는 허브의 송출)
3. 팝업에서 확인 — **소스**(어떤 오디언스인지) / **대상 규모**(몇 명인지) / **환경**(테스트·운영)
4. 전송 실행

### ⚠️ 전송 전 확인
- 운영 전송은 **실제 반영**입니다. 실행 후 취소할 수 없습니다.
- 대상 규모와 소스를 **반드시 확인**한 뒤 실행해 주세요.
- 결과는 **전송 이력**에서 확인합니다.

### 전송 이력
그룹명 · 필터 요약 · 오디언스 수 · 환경 · 전송자 · 일시 · 런컴 ID.
카트 묶음으로 보낸 건은 `카트 묶음: {이름} ({조각 수})` 형태로 표시됩니다.

### 데이터 취급
- 오디언스 데이터는 **런컴 지면 전송 목적으로만** 사용됩니다.
- 제3자 플랫폼으로의 반출은 이루어지지 않습니다.

## 10. AI(ML) 기반 오디언스 — 검증 결과

### 룰 기반 vs ML 기반
| 구분 | 룰 기반 (필터·페르소나) | ML 기반 |
|---|---|---|
| 방식 | 사람이 조건을 지정 | 과거 반응 데이터를 학습해 확률 예측 |
| 장점 | 의도가 명확하고 설명하기 쉬움 | 사람이 찾기 어려운 조합 발굴 |
| 한계 | 아는 조건만 찾을 수 있음 | 학습 데이터에 의존 |

### 4개 모델 검증 결과
기준(baseline) 전환율 **9.54%** 대비:

| 모델 | 규모 | 전환율 | 리프트 | 판정 |
|---|---|---|---|---|
| **반응 예측** | 약 30만 명 | **17.30%** | **1.81×** | ✅ **채택** |
| AI 탐색 | 1,882명 | 11.58% | 1.21× | 보조 |
| 유사확장 v2 | 50만 명 | 7.31% | 0.77× | ❌ 폐기 |
| 유사확장 v1 | 50만 명 | 4.67% | 0.49× | ❌ 폐기 |

- **반응 예측 모델**이 기준 대비 **1.81배** 높은 전환율을 보여 채택되었습니다. 전환자 51,907명을 기준으로 검증했습니다.
- **v1·v2(유사확장) 모델은 기준보다 낮은 성과가 나와 폐기했습니다.** 성과가 나오지 않은 모델을 그대로 쓰지 않고 투명하게 알려드립니다.
- 현재 채택 모델은 **전송 대기** 상태이며, 실제 집행은 **협의 후 진행**합니다.

## 11. 성과 · 인사이트
*(관리자 권한 메뉴)*

### 📊 매체 성과

**상단 요약** — 총 노출 / 총 전환 / 총 광고비 / 활성 매체. 기간은 **7일 / 30일 / 90일** 선택.

**시각화 전환** (우측 상단)
| 옵션 | 특징 | 추천 |
|---|---|---|
| 현재 | 간단한 막대 | 빠른 확인 |
| **정제 2D** | 효율 진단 버블 · 매체 랭킹 · 전환 점유 · 추이 | ⭐ **일상 업무** |
| 🧊 3D | 날짜×매체×노출 입체 지형 | 패턴 탐색 |
| ✦ BizViz | 코스믹 스타일 | 발표·보고용 |

> **「매체 효율 진단」 읽는 법**
> 가로축 = 광고비, 세로축 = 전환율, 원 크기 = 노출량
> → **왼쪽 위의 큰 원** = 적은 비용으로 높은 전환을 낸 매체

**하단 좌측 — 매체 순위**: 매체별 노출·클릭·전환·광고비(최대 50행). 행을 클릭하면 해당 매체의 일별 추이만 표시됩니다. 🔁 배지 = 소재 반응 데이터 보유 매체.

**하단 우측 — 🔁 오디언스 반응 소재 (폐루프)**: 오디언스를 선택하면 그 오디언스가 **실제로 전환한 광고 소재 TOP**을 보여줍니다. 좌측에서 매체를 선택하면 자동으로 필터됩니다.

> 결과가 비어 있어도 정상일 수 있습니다. 적립형 매체(출석체크·적립 등)는 소재명 없이 전환이 기록되어 소재별 분석 대상이 아닙니다. 🔁 배지 매체에서 확인해 주세요.

### 그 외
| 메뉴 | 내용 |
|---|---|
| **소비 트렌드** | 기간별 소비 추이 |
| **카드사 비교** | 카드사별 거래·금액·인기 업종 비교 |
| **쇼핑상품** | 쇼핑 상품 랭킹·상세 |
| **분류 맵핑 관리** | 업종 코드에 표시 라벨을 지정. 지정 내용은 서버에 저장되어 전 사용자 화면에 반영됩니다 |

### 권장 운영 사이클
```
오디언스 정의 → 담기·확정 → 전송 → (집행) → 매체 성과 확인 → 반응 소재 파악 → 다음 오디언스 개선
```

## 12. 자주 묻는 질문

**Q. 좌측 메뉴에 성과·인사이트가 보이지 않습니다.**
광고주 권한에서는 표시되지 않습니다. 필요하시면 담당자에게 권한을 문의해 주세요.

**Q. 페르소나와 카트는 무엇이 다른가요?**
**페르소나**는 자주 쓰는 *조건 세트*(재사용 목적), **카트**는 그 조건들을 *담아 하나의 타겟으로 묶는* 장바구니입니다. 페르소나를 카트에 담아 확정하는 흐름이 일반적입니다.

**Q. 지하철 역이나 멤버십 가맹점으로 타겟을 뽑고 싶습니다.**
현재 해당 축은 **탐색(분석)용**이며 추출에는 반영되지 않습니다. 담기 시 성별·연령·지역만 반영됩니다. 확장이 필요하시면 담당자에게 문의해 주세요.

**Q. 조회가 느립니다.**
대용량 데이터를 처음 조회할 때 최대 1~2분이 걸릴 수 있습니다. 이후 동일 조건은 수 초 내로 응답합니다. 상단 **LIVE 응답속도** 표시로 확인할 수 있습니다.

**Q. 폐루프(반응 소재) 결과가 비어 있습니다.**
정상일 수 있습니다. 적립형 매체는 소재명 없이 전환이 기록됩니다. 🔁 배지가 있는 매체를 선택해 주세요.

**Q. 필터를 걸었는데 대상이 0명입니다.**
조건이 과도하게 좁을 수 있습니다. 조건을 하나씩 해제하며 규모를 확인해 보세요.

**Q. 전송을 취소하고 싶습니다.**
운영 전송은 실행 즉시 반영되어 취소할 수 없습니다. 담당자에게 문의해 주세요.

**Q. 매체 성과 숫자가 다른 리포트와 다릅니다.**
집계 기준일·기간·매체 범위가 다를 수 있습니다. 기간 선택(7/30/90일)을 먼저 확인해 주세요.

**Q. 데이터는 언제 갱신되나요?**
일 단위 배치로 갱신됩니다. 당일 데이터는 익일 반영됩니다.

## 13. 용어 사전

| 용어 | 의미 |
|---|---|
| **오디언스** | 특정 조건을 만족하는 이용자 집합 |
| **페르소나** | 별명 + 필터 세트. 자주 쓰는 타겟 조건을 저장해 재사용 |
| **오디언스 카트** | 여러 조건을 담아 하나의 타겟으로 묶는 장바구니 |
| **타겟** | 카트에서 확정되어 송출 가능한 상태가 된 오디언스 |
| **라벨 / 태그** | 타겟 분류용 표식 (캠페인명·고객명 등 내부 관리 목적) |
| **세그먼트** | 오디언스를 나누는 기준 (성별·연령·업종 등) |
| **큐브** | 빠른 조회를 위해 미리 집계해 둔 데이터 |
| **ADID** | 광고식별자 — 이용자 매칭에 사용 |
| **노출 (Impression)** | 광고가 보여진 횟수 |
| **클릭** | 광고를 클릭한 횟수 |
| **전환 (Conversion)** | 광고 후 실제 행동(구매·참여 등)이 발생한 건수 |
| **전환율** | 전환 ÷ 노출 |
| **CTR** | 클릭 ÷ 노출 |
| **폐루프 (Closed-loop)** | 전송한 오디언스가 실제 어떤 소재에 반응했는지 되돌아보는 분석 |
| **적립형 매체** | 출석체크·포인트 적립 방식 매체 (소재명 없이 전환 기록) |
| **소재형 매체** | 개별 광고 소재로 전환이 기록되는 매체 (🔁 배지) |
| **리프트 (Lift)** | 기준 대비 성과 배수. 1.81× = 기준의 1.81배 |
| **baseline** | 비교 기준이 되는 평균 전환율 (현재 9.54%) |
